# Viewing 도메인 이용 정책

> MOMO CITY 강의 시청, 진척도, 이어보기 기능과 관련된 정책을 안내합니다.
> 이 문서는 사용자가 자주 묻는 질문에 대한 답변 근거로 사용됩니다.

---

## 1. 강의 시청 권한 (누가 볼 수 있나요?)

강의를 시청하려면 기본적으로 **수강 신청(enrollment)** 이 되어 있어야 합니다. 단, 역할에 따라 예외가 있습니다.

| 역할 | 접근 가능 조건 |
|---|---|
| 학생(일반 사용자) | 해당 강의를 **수강 신청한 경우에만** 시청 가능 |
| 강사(TEACHER) | **본인이 개설한 강의**는 수강 신청 없이 시청 가능 |
| 관리자(ADMIN) | 모든 강의를 제한 없이 시청 가능 |

수강 신청이 되어 있지 않은 학생이 강의 시청을 시도하면 접근이 거부됩니다(403).

### Q. 수강 신청 안 한 강의도 미리 볼 수 있나요?
불가능합니다. 학생 계정은 수강 신청이 완료된 강의만 시청할 수 있습니다.

---

## 2. 순차 시청 제한

- 챕터는 순서(orderNo)를 가지며, **1번 챕터는 누구나 바로 시청 가능**합니다.
- 2번 챕터 이후부터는 **바로 이전 챕터를 완료해야만** 접근할 수 있습니다.
- 이전 챕터의 시청 기록 자체가 없거나, 있어도 완료 처리가 안 되어 있으면 다음 챕터 접근이 거부됩니다(403).
- **관리자와 강사는 순차 시청 제한을 받지 않습니다.**

### Q. 3번 챕터를 먼저 듣고 싶은데 왜 막히나요?
2번 챕터를 완료하지 않았기 때문입니다. 순차 시청 정책상 이전 챕터 완료가 선행되어야 다음 챕터에 접근할 수 있습니다.

### Q. 강사인데도 순차 시청 제한이 걸려요.
정상적으로는 강사·관리자 권한이면 제한이 적용되지 않아야 합니다. 계정 권한이 올바르게 설정되어 있는지 확인이 필요합니다.

---

## 3. 영상 스트리밍 URL 발급

- **엔드포인트**: `GET /lectures/{lectureId}/chapters/{chapterId}/stream`
- 발급되는 Presigned URL의 **유효 시간은 1시간(3600초)** 입니다.
- URL 발급 시 수강 여부 확인과 순차 시청 제한이 함께 검증됩니다.
- 발급 응답에는 마지막으로 시청했던 위치(`lastPositionSec`)도 함께 내려가 이어보기에 활용됩니다.

### Q. 발급받은 영상 URL을 1시간 넘게 써도 되나요?
안 됩니다. 1시간이 지나면 URL이 만료되므로 재생 중 만료가 임박하면 다시 발급받아야 합니다.

---

## 4. 진척도 저장 및 챕터 완료 처리

- **엔드포인트**: `PATCH /lectures/{lectureId}/chapters/{chapterId}/progress`
- 프론트엔드는 **약 30초 주기**로 현재 재생 위치(`playbackSeconds`)를 서버에 전송합니다. 이 저장은 **사용자가 별도 버튼을 누르지 않아도 재생 중 자동으로 반복 호출**되는 방식이며, "이어보기" 버튼 클릭 여부와 무관하게 계속 진행됩니다.
- 진척도 판단 기준(도메인 로직):
  - `playbackSeconds`가 기존 `watchedSeconds`보다 **크고**, 그 증가폭이 **30초 이하**인 경우에만 "의미 있는 진척"으로 인정되어 `watchedSeconds`가 갱신됩니다. (전송 주기가 30초이므로, 정상 재생이라면 매 요청마다 최대 30초씩만 늘어나는 것이 자연스럽습니다.)
  - 뒤로 감기(값이 줄어듦)이거나, 30초를 초과해서 앞으로 건너뛴 경우는 스킵으로 간주되어 `watchedSeconds`가 갱신되지 않고 별도로 집계(모니터링)됩니다.
- **영상의 90% 이상을 시청하면 해당 챕터가 "완료" 처리**됩니다.
- 이미 완료된 챕터를 다시 완료 처리해도 완료 이벤트는 **최초 1회만** 발행됩니다(중복 방지).

### Q. 진척도는 몇 초마다 저장되나요?
약 30초 주기로 자동 저장됩니다. 사용자가 특별히 버튼을 누르지 않아도 재생 중이면 계속 저장됩니다.

### Q. 영상을 90%만 봐도 완료로 처리되나요, 100% 다 봐야 하나요?
90% 이상 시청하면 완료로 처리됩니다. 끝까지 보지 않아도 됩니다.

### Q. 같은 챕터를 여러 번 완료 요청하면 포인트도 여러 번 받나요?
아닙니다. 완료 이벤트(및 그에 따른 포인트 지급)는 최초 완료 시 1회만 발생합니다.

---

## 5. 강의실 나가기(마지막 위치 저장)

- **엔드포인트**: `PATCH /lectures/{lectureId}/chapters/{chapterId}/exit`
- 이 API는 **진척도 자동 저장과는 별개의 추가 저장**입니다. 시청 중 강의실을 나갈 때(나가기 확인 팝업 등) 호출되어, 정확한 마지막 재생 위치(`lastPositionSec`)를 별도로 저장합니다.
- 즉, 4번 항목의 30초 주기 자동 저장은 "이어보기/나가기" 버튼과 무관하게 항상 동작하고, 이 API는 사용자가 명시적으로 나갈 때 마지막 위치를 한 번 더 정밀하게 확정하는 역할입니다.
- 저장 방식은 챕터 완료 여부에 따라 달라집니다.

  | 챕터 상태 | 저장되는 값 |
  |---|---|
  | 완료(isCompleted=true) | 요청으로 받은 `lastPositionSec`을 그대로 저장 |
  | 미완료(isCompleted=false) | 실제 시청 위치인 `watchedSeconds` 값으로 저장(당겨서 나간 경우 실제 본 위치로 보정) |

### Q. "이어보기" 버튼을 안 눌러도 진척도가 저장되나요?
네. 진척도는 재생 중 30초 주기로 자동 저장되며, 이는 이어보기/나가기 버튼 클릭 여부와 무관하게 동작합니다. 나가기 API는 마지막 위치를 한 번 더 정밀하게 확정하는 보조 저장일 뿐입니다.

---

## 6. 챕터 완료 시 포인트 지급

- 챕터가 **최초로 완료 처리**될 때 사용자에게 **10포인트**가 지급됩니다.
- 지급 내역은 포인트 원장(order_history)에도 함께 기록됩니다.
- 포인트 지급은 별도 트랜잭션으로 분리되어 처리되므로, 완료 이벤트가 발행된 후 비동기적으로 반영됩니다.

### Q. 포인트는 언제 얼마나 받나요?
챕터를 최초로 완료(90% 이상 시청)하는 순간 10포인트가 지급됩니다. 재시청으로는 추가 지급되지 않습니다.

## 7. 이어보기 / 진척도 조회

| 기능 | 엔드포인트 | 설명 |
|---|---|---|
| 챕터 이어보기 | `GET /lectures/{lectureId}/chapters/{chapterId}/resume` | 마지막 재생 위치 반환, 기록 없으면 0 |
| 전체 진척도 조회 | `GET /lectures/{lectureId}/progress` | 완료 챕터 시간 + 미완료 챕터 시청 시간 합산으로 계산 |
| 챕터별 진척도 조회 | `GET /lectures/{lectureId}/chapters/progress` | 챕터별 시청 시간/완료 여부/접근 가능 여부 반환 |
| 카테고리별 이어보기 | `GET /enrollments/continue-learning` | 카테고리 내 가장 최근에 시청한 강의·챕터 반환 |
| 내 수강 강의 목록 | `GET /user/me/lectures` | 수강 중인 강의 전체 목록(진척도 포함) |

- 전체 진척도(%)는 **완료된 챕터의 전체 재생시간 + 미완료 챕터의 실제 시청시간**을 합산하여 전체 재생시간 대비 비율로 계산됩니다. 시청 시간이 챕터 길이를 초과해서 집계되는 일은 없도록 상한이 걸려 있습니다.
- 계산 결과는 항상 100%를 넘지 않도록 보정됩니다.

### Q. 카테고리별 이어보기에 아무것도 안 뜨는데 오류인가요?
수강 중인 강의가 없거나 해당 카테고리에 시청 기록이 없으면 정상적으로 빈 응답이 내려갑니다. 오류가 아닙니다.

---

## 8. 요약표

| 항목 | 정책 |
|---|---|
| 시청 권한 | 수강 신청자 / 본인 강의 강사 / 관리자만 가능 |
| 순차 시청 | 1번 챕터 제외, 이전 챕터 완료해야 다음 접근(관리자·강사 예외) |
| 스트리밍 URL 유효시간 | 1시간(3600초) |
| 챕터 완료 기준 | 재생 90% 이상 |
| 완료 이벤트 | 최초 1회만 발행(중복 방지) |
| 포인트 지급 | 챕터 최초 완료 시 10포인트 |
| 전체 진척도 계산 | 완료 챕터 시간 + 미완료 챕터 시청 시간 합산, 최대 100% |
| 진도 저장 주기 | 약 30초(버튼 클릭과 무관하게 자동 저장) |

| 자주 헷갈리는 포인트 | 정답 |
|---|---|
| "90% vs 100% 완료 기준" | 90% 이상이면 완료 |
| "재시청 시 포인트 또 받나요?" | 아니요, 최초 완료 시에만 지급 |
| "강사도 순차 시청 제한 걸리나요?" | 아니요, 강사·관리자는 예외 |
| "URL은 계속 재사용 가능한가요?" | 아니요, 1시간 후 만료 |
