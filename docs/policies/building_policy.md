# MomoCity 건물 생성 정책

## 1. 건물은 어떻게 생기나요?

건물을 직접 만드는 별도 API는 없습니다. **수강 신청에 성공하면 부수 효과로 자동 생성**됩니다.

- 엔드포인트: `POST /api/v1/lectures/{lectureId}/enrollments?position={position}`
- 수강 신청이 성공하면, 서버가 내부적으로 "이 자리에 건물을 지어야 하는지" 판단 후 필요시 자동으로 건물을 생성합니다.

## 2. position(위치) 값

- 수강 신청 시 `position` 파라미터로 건물을 세울 위치를 지정합니다.
- **1 ~ 5 사이 값만 허용**됩니다. 이 범위를 벗어나면 요청이 거부됩니다.
- position 값이 아예 없으면(null) "건물 값은 필수입니다." 오류로 거부됩니다.

## 3. 건물 생성 판단 로직

수강 신청한 강의의 카테고리를 기준으로, 아래 순서로 판단합니다.

1. **해당 position에 이미 건물이 있는 경우**
   - 카테고리가 같으면 → 새로 안 짓고 그냥 수강 신청만 진행 (중복 생성 안 됨)
   - 카테고리가 다르면 → **거부**("이미 다른 건물이 생성되어 있습니다.")
2. **해당 position은 비어있지만, 같은 카테고리 건물을 다른 위치에 이미 가지고 있는 경우**
   - 새로 짓지 않고 수강 신청만 진행 (한 유저는 카테고리당 건물을 1개만 보유)
3. **둘 다 아닌 경우**
   - 새 건물 생성 (레벨 1로 시작)

즉, **한 사용자는 카테고리별로 건물을 딱 1개씩만 가질 수 있고**, 같은 카테고리 강의를 여러 개 수강해도 건물이 여러 개 생기지 않습니다.

## 4. 건물 조회

| 기능 | 엔드포인트 |
|---|---|
| 내 건물 목록(메인페이지용) | `GET /api/v1/user/buildings` |
| 친구 건물 목록(친구 도시 방문) | `GET /api/v1/user/{userId}/buildings` |

## 5. 건물 정보

- 필드: 카테고리(`category`), 위치(`position`), 레벨(`level`), 생성/수정 시각
- **새로 생성된 건물은 항상 레벨 1**로 시작합니다.
