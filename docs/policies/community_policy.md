# Community 도메인 이용 정책

> MOMO CITY 커뮤니티(게시글/댓글/대댓글/좋아요) 기능과 관련된 정책을 안내합니다.
> 이 문서는 사용자가 자주 묻는 질문에 대한 답변 근거로 사용됩니다.

---

## 1. 접근 권한 — 로그인 여부로만 구분

커뮤니티 이용 권한은 **회원 등급이 아니라 로그인(토큰) 여부 하나로만 구분**됩니다. 로그인만 되어 있으면(BASIC 등급 이상) 등급에 상관없이 게시글 작성, 좋아요, 댓글/대댓글 작성을 **모두** 이용할 수 있습니다.

| 구분 | 이용 가능 기능 |
|---|---|
| **비로그인(토큰 없음)** | 게시글 목록/검색/상세 조회, 댓글·대댓글 조회만 가능 |
| **로그인(BASIC 이상, 등급 무관)** | 조회 기능 전부 + **게시글 작성**, **좋아요**, **댓글/대댓글 작성** 모두 가능 |

- 게시글 목록·상세 조회는 비로그인 상태에서도 가능하도록 설계되어 있습니다(인증 정보가 없어도 조회 API가 동작).
- 쓰기 작업(게시글 작성/좋아요/댓글)은 로그인만 되어 있으면 이용 가능하며, PLUS/PRO 같은 별도 등급 제한은 없습니다.

### Q. 로그인 안 해도 게시글을 볼 수 있나요?
네, 목록/검색/상세 조회는 비로그인 상태에서도 가능합니다.

### Q. 아무나 게시글을 쓸 수 있나요?
로그인한 회원이면 등급과 상관없이 게시글 작성이 가능합니다. 별도의 상위 등급 제한은 없습니다.

## 2. 게시글 작성 흐름

게시글 작성은 아래 순서로 진행됩니다.

1. **게시글 생성** — `POST /api/v2/posts` (제목, 카테고리, 썸네일 URL로 게시글 뼈대 생성)
2. **이미지 업로드(선택)** — `POST /api/v2/posts/images` (본문에 들어갈 이미지를 S3에 업로드하고 URL 발급)
3. **콘텐츠 업로드** — `POST /api/v2/posts/{postId}/contents` (실제 본문 내용을 게시글에 채워 넣음)

- 게시글 콘텐츠 수정은 `PUT /api/v2/posts/{postId}/contents`로 **전체 교체** 방식입니다. 일부만 수정해도 콘텐츠 목록 전체를 다시 보내야 합니다.
- 제목/카테고리만 별도로 수정하려면 `PATCH /api/v2/posts/{postId}`를 사용합니다.

### Q. 게시글 작성할 때 왜 API를 두세 번 나눠 호출해야 하나요?
게시글 뼈대(제목/카테고리) 생성과 이미지 업로드, 본문 콘텐츠 업로드가 각각 분리되어 있기 때문입니다. 이미지가 없는 글이라면 이미지 업로드 단계는 생략할 수 있습니다.

### 썸네일 자동 지정 규칙

콘텐츠 업로드(또는 수정) 시 썸네일이 아래 우선순위로 결정됩니다.

1. **썸네일 URL을 직접 지정한 경우** → 그 값을 그대로 사용
2. **썸네일 미지정 + 본문에 이미지(IMAGE 타입) 콘텐츠가 있는 경우** → **오류.** "이미지를 업로드했다면 썸네일을 지정해주세요" 응답과 함께 요청이 거부됩니다. 이미지가 있는 글은 썸네일을 반드시 직접 골라야 합니다.
3. **썸네일 미지정 + 이미지 콘텐츠도 없는 경우** → 카테고리 기준 **기본 썸네일이 자동으로 지정**됩니다.

### Q. 이미지 넣은 글인데 썸네일을 안 정했더니 오류가 나요.
정상 동작입니다. 본문에 이미지가 있는 글은 썸네일을 직접 지정해야 합니다. 이미지가 아예 없는 글이라면 썸네일을 안 정해도 카테고리 기본 이미지가 자동으로 붙습니다.

### 콘텐츠 작성 시 검증 규칙

| 검증 항목 | 규칙 |
|---|---|
| 이미지 개수 | 게시글 하나당 **IMAGE 타입 콘텐츠 최대 5개**까지, 초과 시 거부 |
| TEXT 콘텐츠 | `content` 값이 비어있으면 거부 |
| IMAGE 콘텐츠 | `imageUrl` 값이 비어있으면 거부 |

이 검증은 최초 업로드(`POST /contents`)와 수정(`PUT /contents`) 양쪽에 동일하게 적용됩니다.

### Q. 이미지를 몇 장까지 넣을 수 있나요?
게시글 하나에 이미지는 최대 5장까지 넣을 수 있습니다.

### 게시글 목록 캐시

게시글 목록 조회(`GET /posts`)는 카테고리·커서·size 조합으로 Redis에 캐시됩니다. 게시글을 새로 작성/수정/삭제하면 **목록 캐시 전체가 한 번에 초기화**됩니다(특정 카테고리만 부분 초기화하는 방식이 아닙니다).

## 3. 게시글 수정/삭제 권한

- 게시글 수정, 콘텐츠 수정, 삭제는 **작성자 본인만** 가능합니다.
- 게시글 삭제는 소프트 삭제로 처리되며, 삭제된 게시글에는 이후 댓글/대댓글을 새로 작성할 수 없습니다.
- 삭제된 게시글/댓글은 즉시 완전히 사라지지 않고, **삭제 처리된 시점으로부터 6개월이 지나면 완전히 삭제**되어 복구할 수 없습니다.

### Q. 삭제된 게시글에 댓글을 달 수 있나요?
불가능합니다. 삭제된 게시글에는 댓글/대댓글 작성이 모두 차단됩니다.

### Q. 삭제한 글이나 댓글을 다시 복구할 수 있나요?
삭제 후 6개월 이내라면 보관 중이지만, 사용자가 직접 복구를 요청할 수 있는 기능은 없으며 6개월이 지나면 완전히 삭제되어 복구가 불가능합니다.

## 4. 댓글 / 대댓글 정책

| 항목 | 정책 |
|---|---|
| 댓글 구조 | 댓글(최상위) → 대댓글(1단계까지만) |
| 대댓글에 대댓글 | ❌ 불가능(2단계 이상 중첩 금지) |
| 삭제된 게시글에 댓글 작성 | ❌ 불가능 |
| 삭제된 댓글에 대댓글 작성 | ❌ 불가능 |
| 댓글/대댓글 삭제 | 작성자 본인만 가능(소프트 삭제) |
| 조회 방식 | 커서 기반 페이지네이션 |

- 댓글 목록 조회 시 각 댓글에 딸린 대댓글은 **최대 5개까지만 우선 노출**되고, 5개를 초과하면 `hasMoreReplies=true`와 함께 별도 대댓글 조회 API(`GET /posts/{postId}/comments/{commentId}/replies`)로 나머지를 조회해야 합니다.
- 댓글/대댓글 응답에는 작성자가 **본인인지 여부**, **게시글 작성자인지 여부**가 함께 내려가 UI에서 "작성자" 뱃지 등을 표시할 수 있습니다.

### Q. 대댓글에 답글을 달았는데 왜 안 달리나요?
정책상 대댓글에는 답글(대댓글의 대댓글)을 달 수 없습니다. 최상위 댓글에만 대댓글을 작성할 수 있습니다.

### Q. 댓글 밑에 대댓글이 5개 넘게 있는데 일부만 보여요.
정상입니다. 댓글 목록 조회 시 대댓글은 최대 5개까지만 함께 내려가며, 그 이상은 대댓글 전용 조회 API로 추가 조회해야 합니다.

## 5. 알림(이벤트) 발생 정책

댓글/대댓글 작성 시 아래 조건에서만 알림 이벤트가 발생합니다. **본인에게 알림이 가는 상황은 제외**됩니다.

| 상황 | 알림 발생 여부 |
|---|---|
| 남이 내 게시글에 댓글을 작성 | ✅ 발생 |
| 내가 내 게시글에 댓글을 작성(자기 글에 자기 댓글) | ❌ 발생 안 함 |
| 남이 내 댓글에 대댓글을 작성 | ✅ 발생 |
| 내가 내 게시글 또는 내 댓글에 대댓글을 작성 | ❌ 발생 안 함 |

### Q. 제 글에 제가 댓글을 달았는데 알림이 안 와요.
정상입니다. 본인 게시글에 본인이 댓글을 다는 경우는 알림이 발생하지 않도록 설계되어 있습니다.

## 6. 좋아요 정책

| 기능 | 엔드포인트 |
|---|---|
| 좋아요 | `POST /api/v2/posts/{postId}/likes` |
| 좋아요 취소 | `DELETE /api/v2/posts/{postId}/likes` |
| 좋아요 목록 조회 | `GET /api/v2/posts/{postId}/likes` |

- 좋아요/좋아요 취소 응답에는 최신 좋아요 수(`likeCount`)와 현재 사용자의 좋아요 여부(`isLiked`)가 함께 내려와, 클라이언트가 별도 재조회 없이 화면을 갱신할 수 있습니다.
- **중복 좋아요는 차단됩니다.** 이미 좋아요를 누른 게시글에 다시 좋아요를 요청하면 "이미 좋아요를 눌렀습니다" 오류가 반환됩니다.
- **좋아요를 누르지 않은 상태에서 취소를 요청하면** "좋아요를 누르지 않았습니다" 오류가 반환됩니다.
- **삭제된 게시글에는 좋아요를 누를 수 없습니다.**
- **본인 게시글에 본인이 좋아요를 눌러도 알림은 발생하지 않습니다**(댓글/대댓글과 동일한 원칙).

### Q. 좋아요를 두 번 누르면 두 번 카운트되나요?
아니요, 이미 누른 상태에서 다시 요청하면 오류가 나고 중복 카운트되지 않습니다.

### Q. 안 눌렀는데 좋아요 취소를 하면 어떻게 되나요?
오류가 발생합니다. 좋아요를 먼저 누른 적이 있어야 취소할 수 있습니다.

## 7. 게시글 조회/검색 정책

| 기능 | 엔드포인트 | 비고 |
|---|---|---|
| 게시글 목록 | `GET /posts?category=&cursor=&size=` | 카테고리 필터, 커서 페이지네이션 |
| 게시글 상세 | `GET /posts/{postId}` | 비로그인도 조회 가능 |
| 내 게시글 목록 | `GET /posts/me` | 로그인 필요 |
| 상대방 게시글 목록 | `GET /posts/users/{targetUserId}` | 본인 ID로 요청 시 거부(마이페이지 이용 안내) |
| 내 대시보드 | `GET /posts/me/dashboard` | 내 게시글 통계 |
| 상대방 대시보드 | `GET /posts/users/{targetUserId}/dashboard` | 본인 ID로 요청 시 거부 |
| 게시글 검색 | `GET /posts/search?keyword=` | 키워드 **2자 이상** 필수 |
| 연관 게시글 추천 | `GET /posts/{postId}/recommendations` | 같은 카테고리 인기글 + 작성자의 다른 글 |

- **검색 키워드는 공백만 입력하거나 1자 이하로는 검색할 수 없습니다.** 최소 2자 이상 입력해야 합니다.
- 상대방 게시글 목록/대시보드 조회 시 대상 ID가 본인과 같으면 "본인 페이지는 마이페이지에서 확인해주세요" 안내와 함께 요청이 거부됩니다.

### Q. 검색어를 한 글자만 넣었는데 왜 오류가 나나요?
검색 키워드는 최소 2자 이상이어야 합니다. 1자 이하 또는 공백은 허용되지 않습니다.

### Q. 제 프로필 페이지를 상대방 조회 API로 볼 수 있나요?
불가능합니다. 본인 ID로 상대방 조회 API를 호출하면 거부되며, 본인 정보는 `/posts/me`, `/posts/me/dashboard`로 조회해야 합니다.

### 조회수 증가 방식

게시글 상세 조회(`GET /posts/{postId}`) 시 조회수는 **비동기로 증가**합니다. 상세 정보 응답과 조회수 증가 처리가 별도 스레드에서 동시에 진행되므로, 조회수 반영이 응답 속도에 영향을 주지 않습니다. 다만 그 말은 즉, **방금 막 조회한 시점의 응답에는 이번 조회로 늘어난 수치가 반영되지 않을 수도 있다**는 뜻입니다(다음 조회부터 반영).

### 연관 게시글 추천 방식

추천 게시글은 두 그룹으로 구성됩니다.

| 그룹 | 조건 | 개수 |
|---|---|---|
| 인기글 | 현재 글과 **같은 카테고리**의 인기 게시글(현재 글 제외) | 최대 3개 |
| 작성자의 다른 글 | **같은 작성자**가 쓴 최신 게시글(현재 글 및 위 인기글과 중복 제외) | 최대 2개 |

### 대시보드 통계 항목

내 대시보드(또는 상대방 대시보드)에는 아래 네 가지 통계가 함께 집계되어 내려갑니다.

- 전체 게시글 수
- 전체 조회수 합계
- 전체 좋아요 수 합계
- 전체 댓글 수 합계(대댓글 제외, 게시글들에 달린 최상위 댓글만 집계)

| 게시글 목록 캐시 무효화 범위 | 작성/수정/삭제 시 목록 캐시 전체 초기화 |
